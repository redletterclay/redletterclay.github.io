## Critical Rules (enforce these every time)

### 1. Lexical `as const` — ALWAYS required

In any Lexical node object (seed scripts, test data, anywhere), `format` and `direction` must use `as const`:

```ts
// CORRECT — paragraph/heading nodes
{ type: 'paragraph', format: '' as const, direction: 'ltr' as const, ... }

// CORRECT — text nodes (numeric format values need as const too)
{ type: 'text', format: 0 as const, mode: 'normal' as const, ... }
{ type: 'text', format: 1 as const, mode: 'normal' as const, ... }  // bold

// WRONG — TypeScript widens '' → string, 0/1 → number; both break Payload's union types
{ type: 'paragraph', format: '', direction: 'ltr', ... }
{ type: 'text', format: 0, mode: 'normal', ... }
```

This applies to **every** node in the tree: root, paragraph, heading, text, list, listitem, etc.

### 2. Keep seed scripts in sync with field types

When a field changes to Lexical `richText`, update **all** seed scripts that write to that field. Check `scripts/` for any that reference the field by name.

### 3. Never edit `payload-types.ts`

This file is auto-generated by `npx payload generate:types`. Always regenerate it rather than editing it.

### 4. Migrations are auto-generated

Run `npx payload migrate:create` to generate a migration after schema changes. Never write migration SQL by hand unless debugging.

## Standard Lexical Node Structure

Use this shape for seed data and test fixtures:

```ts
const richTextValue = {
  root: {
    type: 'root' as const,
    format: '' as const,
    indent: 0,
    version: 1,
    direction: 'ltr' as const,
    children: [
      {
        type: 'paragraph',
        format: '' as const,
        indent: 0,
        version: 1,
        direction: 'ltr' as const,
        textFormat: 0,
        textStyle: '',
        children: [
          {
            type: 'text',
            format: 0 as const,
            style: '',
            mode: 'normal' as const,
            detail: 0,
            text: 'Your text here',
            version: 1,
          },
        ],
      },
    ],
  },
}
```

For **bold** text, set `format: 1 as const` on the text node.

> **Note:** `as const` is required on numeric format values too — `format: 0 as const`, `format: 1 as const`. Bare `0` or `1` widens to `number`, which fails Payload's union type just like bare `''` does.

## Access Control Patterns

Three access functions cover most use cases:

```ts
import { anyone } from '../access/anyone'
import { authenticated } from '../access/authenticated'
import { authenticatedOrPublished } from '../access/authenticatedOrPublished'

// Public read, authenticated write (most collections)
access: {
  read: anyone,
  create: authenticated,
  update: authenticated,
  delete: authenticated,
}

// Drafts pattern — public sees only published docs
access: {
  read: authenticatedOrPublished,
  // ...
}
```

## ISR Revalidation Pattern

All collections and globals that affect the frontend must revalidate Next.js routes in `afterChange`/`afterDelete` hooks:

```ts
import { revalidatePath } from 'next/cache'

const afterChange: CollectionAfterChangeHook = ({ doc, context }) => {
  if (!context.disableRevalidate) {
    revalidatePath('/your-path')
    revalidatePath('/your-path/[slug]', 'page')
  }
  return doc
}
```

The `!context.disableRevalidate` guard prevents revalidation during seed scripts (pass `{ disableRevalidate: true }` in the request context when seeding).

## Running Seed Scripts

```bash
# Example — always pass PAYLOAD_SECRET and DATABASE_URL
PAYLOAD_SECRET=redletterclay \
DATABASE_URL=postgresql://payload:password@localhost:5432/payload \
npx tsx scripts/seed-stockists.ts
```

When writing seed scripts:
- Import and use the Payload local API (`getPayload({ config })`)
- Pass `{ overrideAccess: true }` so seeds bypass access control
- Pass `context: { disableRevalidate: true }` to skip ISR during seeding

## Migrations

```bash
# After schema change
npx payload migrate:create

# Apply pending migrations
npx payload migrate

# Rollback last migration
npx payload migrate:down
```

Migrations live in `src/migrations/` and are auto-generated. Commit both the `.ts` and the `.json` snapshot files.

## Media & ImageKit

The `Media` collection auto-syncs uploads to ImageKit. Key behavior:
- Uploaded files get an `imagekitUrl` field set by an `afterChange` hook
- The `afterRead` hook replaces `doc.url` with `imagekitUrl` for frontend use
- A serial upload queue (500ms gap, 5 retries) prevents ImageKit rate limiting
- Image sizes generated: `thumbnail`, `square`, `small`, `medium`, `large`, `xlarge`, `og`

When referencing media in seed scripts, use the Payload local API to create Media docs first, then reference the returned `id`.

## Common Patterns to Know

### Adding a new collection field

1. Add the field to the collection file
2. Run `npx payload migrate:create` to generate the migration
3. Update any seed scripts if the field is required or needs data
4. Run `npx payload generate:types` to refresh `payload-types.ts`

### Adding a new global

1. Create `src/YourThing/config.ts` (follow existing globals as a template)
2. Add `revalidatePath()` in `afterChange` hook
3. Register it in `payload.config.ts` under `globals`
4. Generate types

### Lexical features available (defaultLexical)

`defaultLexical` (used for simple fields like Media caption):
- Bold, Italic, Underline, Link, Paragraph

Richer fields (Posts `bodyRich`, Products `description`, Announcement `info`) add:
- HeadingFeature (h1–h4, context-dependent)
- BlocksFeature (Banner, Code, MediaBlock, or others)
- FixedToolbarFeature, InlineToolbarFeature, HorizontalRuleFeature
